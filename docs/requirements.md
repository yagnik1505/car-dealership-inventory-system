# Project Requirements

## Overview

The Car Dealership Inventory System is a full-stack web application that allows customers to browse and purchase vehicles while enabling administrators to manage dealership inventory securely.

---

## Actors

### Customer
- Register and login
- View all vehicles
- Search vehicles
- Purchase vehicles

### Administrator
- Add vehicles
- Update vehicle details
- Delete vehicles
- Restock inventory

---

## Functional Requirements

### Authentication
- User registration
- User login
- JWT-based authentication

### Vehicle Management
- Add vehicle
- View all vehicles
- Update vehicle
- Delete vehicle
- Search by make, model, category, and price range

### Inventory
- Purchase vehicle
- Reduce stock after purchase
- Prevent purchase when stock is unavailable
- Restock inventory (Admin only)

---

## Non-Functional Requirements

- Secure authentication
- Clean and maintainable code
- RESTful API design
- Test-Driven Development (TDD)
- Meaningful Git commit history
